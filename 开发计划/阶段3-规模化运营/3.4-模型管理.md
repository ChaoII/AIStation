# 阶段3：规模化运营 — 模型管理

## 1. 模型存储

### 1.1 目录结构

```
data/models/
├── intrusion/
│   ├── v1.0.0/
│   │   ├── engine.onnx
│   │   └── metadata.json        # 训练信息、mAP、标签映射
│   └── v2.0.0/
│       ├── engine.onnx
│       └── metadata.json
├── face_detect/
│   ├── v1.0.0/
│   │   ├── det.onnx
│   │   ├── rec.onnx
│   │   └── metadata.json
│   └── v2.0.0/...
└── symlinks/
    ├── intrusion → ../intrusion/v2.0.0/
    └── face_detect → ../face_detect/v1.0.0/
```

### 1.2 模型版本配置

`AlgorithmModel` 的 `runtime_config` 中增加版本字段：

```json
{
  "engine": "onnxruntime",
  "gpu": {"enabled": true, "device_id": 0},
  "model_version": "v2.0.0",
  "model_symlink": "data/models/symlinks/intrusion"
}
```

模型路径支持三种模式：

| 模式 | 说明 | 更新方式 |
|------|------|----------|
| 绝对路径 | 直接指向具体模型文件 | 编辑算法 |
| 符号链接 | 指向 symlinks 目录，切换版本只需改链接 | 运维操作 |
| 版本号 | `model_version` + 算法类型自动解析 | 编辑算法 |

## 2. 模型热更新

### 2.1 热更新流程

```
运维准备新模型
  │
  ▼
1. 上传模型文件到 data/models/{type}/v{new_version}/
2. 更新符号链接: ln -sfn ../intrusion/v2.0.0 symlinks/intrusion
3. 热更新通知：
   POST /api/v1/algorithm/model/hot-reload
   {"algorithm_type": "INTRUSION", "version": "v2.0.0"}
  │
  ▼
Web 端：
  1. 更新 AlgorithmModel 的 model_path
     （如果使用符号链接，实际路径不变，无需更新 DB）
  2. 通知相关 AI 盒子：将命令 {"action": "reload_model", "task_id": ...} 放入命令队列
  │
  ▼
AI 盒子 Agent：
  1. 收到 reload_model 命令
  2. stop_inference(task_id)  # 停止当前 Worker
  3. start_inference(task_id)  # 重新启动（加载新模型）
```

### 2.2 Worker 侧热加载（优化版）

Worker 支持 SIGHUP 信号触发模型重载：

```python
# worker.py
model = None

def reload_model(signum, frame):
    global model
    logger.info("收到 SIGHUP，重新加载模型")
    model = create_model(config)

signal.signal(signal.SIGHUP, reload_model)
```

这样 Worker 进程可以不重启，只重新加载模型文件：
```python
proc.send_signal(signal.SIGHUP)  # 替代 SIGTERM + restart
```

## 3. A/B 测试

### 3.1 概念

将部分摄像头的推理流量切换到新模型版本，对比效果：

| 版本 | 摄像头组 | 任务数 | 状态 |
|------|----------|--------|------|
| intrusion/v1.0.0 (当前) | 仓库_东, 仓库_西 | 10 | 稳定 |
| intrusion/v2.0.0 (测试) | 仓库_门口 | 1 | A/B 测试中 |

### 3.2 实现方式

创建两条算法记录，指向不同版本：

```python
# Algorithm A — 生产版本 v1.0.0
AlgorithmModel(name="入侵检测-v1", model_path="data/models/symlinks/intrusion")

# Algorithm B — 测试版本 v2.0.0
AlgorithmModel(name="入侵检测-v2", model_path="data/models/intrusion/v2.0.0/engine.onnx")
```

将少量摄像头关联到 Algorithm B，对比告警准确率。

### 3.3 对比指标

| 指标 | 说明 |
|------|------|
| 检出率 | 每路每天告警数 |
| 误报率 | 人工标记 FALSE_ALARM 占比 |
| 推理延迟 | P50/P95/P99 |
| GPU 利用率 | 推理资源消耗 |

## 4. 模型上传 API

```
POST /api/v1/algorithm/model/upload
Content-Type: multipart/form-data

Parameters:
  algorithm_type: str       # INTRUSION等
  version: str              # v2.0.0
  file: File                # .onnx / .engine / .mnn
  metadata: str (JSON)      # 训练信息、评估指标
```

处理逻辑：
1. 创建 `data/models/{type}/{version}/` 目录
2. 保存模型文件
3. 保存 metadata.json
4. 更新符号链接（若标记为 `set_active=true`）
5. 返回模型路径
