# 阶段3：规模化运营 — 负载均衡策略

## 概述

当有多个 AI 盒子可用时，系统需要智能选择最优盒子来运行新的算法任务。

## 1. 盒子资源模型

每个盒子上报的资源数据（通过心跳）：

```json
{
  "resource_usage": {
    "gpu_util_percent": 65,
    "gpu_mem_used_mb": 4096,
    "gpu_mem_total_mb": 8192,
    "cpu_percent": 45,
    "memory_used_mb": 8192,
    "memory_total_mb": 16384,
    "running_tasks": 2,
    "max_tasks": 4
  }
}
```

## 2. 调度算法

### 2.1 最少任务数（默认）

```python
def select_box_least_tasks(boxes: list[AiBoxModel]) -> AiBoxModel | None:
    """选择当前运行任务数最少的盒子"""
    available = [b for b in boxes if b.status == "ONLINE"
                 and b.running_tasks < b.max_tasks]
    if not available:
        return None
    return min(available, key=lambda b: b.running_tasks)
```

### 2.2 GPU 利用率最低

```python
def select_box_lowest_gpu(boxes: list[AiBoxModel]) -> AiBoxModel | None:
    """选择 GPU 利用率最低的盒子"""
    available = [b for b in boxes if b.status == "ONLINE"
                 and b.running_tasks < b.max_tasks]
    if not available:
        return None
    return min(available, key=lambda b: b.get_gpu_util() or 100)
```

### 2.3 显存最优适配

```python
def select_box_best_fit(boxes: list[AiBoxModel], model_memory_mb: int) -> AiBoxModel | None:
    """选择显存最匹配（刚好能放下模型，不浪费）"""
    available = [b for b in boxes if b.status == "ONLINE"
                 and b.running_tasks < b.max_tasks]
    for b in sorted(available, key=lambda b: b.get_gpu_mem_free()):
        if b.get_gpu_mem_free() >= model_memory_mb:
            return b
    return None
```

### 2.4 标签亲和性

支持盒子打标签，任务按标签调度：

```python
# 盒子标签
box.tags = ["gpu:t4", "location:beijing", "group:production"]

# 任务要求
task.required_tags = ["gpu:t4"]

def select_box_by_tags(boxes: list[AiBoxModel], required_tags: list[str]) -> AiBoxModel | None:
    candidates = [
        b for b in boxes if b.status == "ONLINE"
        and b.running_tasks < b.max_tasks
        and all(tag in (b.tags or []) for tag in required_tags)
    ]
    return min(candidates, key=lambda b: b.running_tasks) if candidates else None
```

## 4. 任务迁移

当盒子过载或故障时，将任务迁移到其他盒子：

```python
async def migrate_task(task_id: int, from_box_id: int, to_box_id: int):
    """迁移任务到指定盒子"""
    # 1. 停原盒子任务
    await send_command(from_box_id, {"action": "stop_task", "task_id": task_id})
    # 2. 等待停止确认（最多 30s）
    await wait_for_status(task_id, "STOPPED", timeout=30)
    # 3. 更新分配记录
    await AiBoxTaskCRUD.update_by_task(task_id, {"box_id": to_box_id, "local_status": "PENDING"})
    # 4. 目标盒子下次心跳收到 start_task 命令
```

### 自动迁移触发条件

| 条件 | 动作 |
|------|------|
| 盒子 OFFLINE > 5min | 将该盒子的 RUNNING 任务迁移到其他 ONLINE 盒子 |
| GPU 利用率 > 95% 持续 10min | 将部分任务迁移到低负载盒子 |
| 盒子维护模式 | 手动触发提前迁移 |
