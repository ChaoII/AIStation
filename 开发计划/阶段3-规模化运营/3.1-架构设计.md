# 阶段3：规模化运营 — 架构设计

## 目标

在阶段2 分布式盒子的基础上，实现：
- 多盒子集群的统一管理和调度
- 负载均衡：按 GPU 利用率、显存、任务数智能分配
- 高可用：主站 HA、盒子故障自动迁移
- 可观测：Prometheus 指标、集中日志
- 模型全生命周期管理

## 核心组件

| 组件 | 说明 | 技术选型 |
|------|------|----------|
| **Nginx** | 负载均衡、SSL 终止、WebSocket 代理 | Nginx / OpenResty |
| **Web 主站** | 无状态水平扩展 | FastAPI + Uvicorn（多 worker） |
| **PostgreSQL** | 主从复制，Patroni 自动故障转移 | Patroni + PG 16 |
| **Redis** | 主从 + Sentinel | Redis 7 |
| **Prometheus** | 指标采集 | prometheus-client |
| **Grafana** | 监控面板 | Grafana |
| **Loki** | 日志聚合 | Grafana Loki + Promtail |
| **AI 盒子集群** | 按机房/GPU 类型分组 | Box Agent（阶段2） |

## 阶段3 新增/变更总览

| 模块 | 变更 |
|------|------|
| Redis 命令队列 | 新增 Redis Streams 存储待执行命令（替代 DB 轮询） |
| 调度器 | 新增负载均衡调度器（资源感知） |
| AI 盒子 | 支持分组、标签、自动发现 |
| 模型管理 | 新增模型存储、版本控制、热更新 |
| 监控 | Prometheus 指标、Loki 日志 |
| 高可用 | Web 无状态化、盒子故障转移 |
