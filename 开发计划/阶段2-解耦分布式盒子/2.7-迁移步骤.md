# 阶段2：解耦分布式盒子 — 从阶段1迁移的步骤

## 概述

从单机架构（阶段1）平滑迁移到分布式盒子架构（阶段2）。核心原则：**Web 主站代码变更最小化**，Worker 代码大部分复用。

## 迁移步骤

### 步骤1：后台新增数据模型

```bash
# 1. 创建 AiBoxModel 和 AiBoxTaskModel
# 2. 添加字段到 video_algorithm_tasks（如果 box_id 不通过独立表）
```

**执行：**
```python
# 在 algorithm/model.py 中添加 AiBoxModel、AiBoxTaskModel
# 运行 alembic 迁移
uv run main.py revision --env=dev
uv run main.py upgrade --env=dev
```

### 步骤2：拆分 Worker 代码

阶段1 的 `worker.py` 在 Web 服务器上作为子进程运行。阶段2 需要：

1. 将 `worker.py` 复制到 AI 盒子 Agent 项目（`/opt/ai-box-agent/worker.py`）
2. 将 `worker.py` 的依赖打包为 `requirements-worker.txt`：

```txt
modeldeploy>=1.7.0
opencv-python-headless>=4.8
httpx>=0.25
numpy>=1.24
```

3. **保持一致**：Worker 的逻辑（拉流 → 推理 → 回调）在两个阶段完全一致

### 步骤3：添加 Agent API 端点

在 `controller.py`（或新建 `agent_controller.py`）中添加：

- `POST /algorithm/agent/register`
- `POST /algorithm/agent/heartbeat`
- `GET/POST/PUT/DELETE /algorithm/box/*`（盒子 CRUD）

**注册路由：**
```python
# module_video/__init__.py
from .algorithm.agent_controller import AgentRouter
video_router.include_router(AgentRouter)
```

### 步骤4：修改布控流程

在 `algorithm/service.py` 的 `create_task_service` / `update_task_service` 中：

```python
# 如果指定了 box_id，同步创建 AiBoxTaskModel
if data.box_id:
    await AiBoxTaskCRUD(auth).create(data=AiBoxTaskCreateSchema(
        algorithm_task_id=new_task.id,
        box_id=data.box_id,
        task_config=build_task_config(new_task, camera, algorithm),
    ))
```

### 步骤5：添加心跳超时检测

在 `init_app.py` lifespan 中添加：

```python
from app.api.v1.module_video.algorithm.box_scheduler import check_box_heartbeats
asyncio.create_task(check_box_heartbeats())
```

### 步骤6：部署 AI 盒子 Agent

在每台 AI 盒子上：

```bash
# 1. 安装 Python 和依赖
apt install python3 python3-pip
pip3 install -r /opt/ai-box-agent/requirements-worker.txt

# 2. 配置 config.yaml
# 修改 register_url、heartbeat_url、secret_key

# 3. 安装 systemd 服务
cp ai-box-agent.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable ai-box-agent
systemctl start ai-box-agent

# 4. 验证
journalctl -u ai-box-agent -f
```

### 步骤7：迁移现有任务

Web 端手动或批量将已有算法任务分配到 AI 盒子：

```python
# 批量迁移脚本
async def migrate_tasks_to_box(box_id: int, task_ids: list[int]):
    for tid in task_ids:
        # 如果任务已启动，先停止
        await stop_inference(tid)
        # 分配到盒子
        task_config = build_task_config(tid)
        await AiBoxTaskCRUD.create(AlgorithmTaskId=tid, box_id=box_id, task_config=task_config)
        # 更新 task.box_id
        await AlgorithmTaskCRUD.update(id=tid, data={"box_id": box_id})
```

## 兼容性保证

| 场景 | 阶段2 行为 | 兼容阶段1？ |
|------|-----------|------------|
| 任务未指定 box_id | 使用本地 Worker（阶段1 模式） | ✅ 兼容 |
| 任务指定 box_id | 由 Agent 管理 | ✅ 新增行为 |
| 既有本地运行任务 + 新分配盒子任务 | 混合运行 | ✅ 兼容 |
| 本地 Scheduler + Agent Scheduler 共存 | 互不干扰 | ✅ 兼容 |

## 回滚方案

如果分布式盒子方案出现问题，可以快速回退到阶段1：

```bash
# 1. 停止 Agent
systemctl stop ai-box-agent

# 2. Web 端将所有盒子任务改为本地运行
UPDATE video_algorithm_tasks SET box_id = NULL WHERE box_id IS NOT NULL;
DELETE FROM video_ai_box_tasks;

# 3. 恢复本地推理
# 通过 Web UI 手动启动推理，或等待调度器拾取
```
