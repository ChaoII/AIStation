# 阶段2：解耦分布式盒子 — 离线韧性机制

## 概述

AI 盒子部署在弱网环境（跨网段/公网），网络中断是常态而非异常。离线韧性确保：
- **网络断开时**：推理任务不受影响，持续运行
- **Agent 重启时**：自动恢复所有任务
- **重连后**：状态同步、结果补传

## 1. 本地持久化（Local Store）

### 存储目录

```
/opt/ai-box-agent/tasks/
├── task_1.json        # 任务 ID 为文件名
├── task_2.json
└── task_3.json
```

### 存储内容

每个 `task_*.json` 包含 Worker 启动所需的完整配置：

```json
{
  "task_id": 1,
  "camera_id": 24,
  "algorithm_type": "INTRUSION",
  "stream_url": "http://zlm/live/camera_24.live.flv",
  "model_path": "/data/models/intrusion/engine.onnx",
  "runtime_config": {...},
  "preset_params": {...},
  "callback_url": "https://web.example.com/api/v1/algorithm/detection/callback",
  "callback_token": "...",
  "fps_target": 5,
  "alarm_interval": 30,
  "snapshot_dir": "/opt/ai-box-agent/snapshots/"
}
```

### 操作接口

```python
class LocalStore:
    def __init__(self, tasks_dir: str):
        self.tasks_dir = Path(tasks_dir)
        self.tasks_dir.mkdir(parents=True, exist_ok=True)

    def save(self, task_id: int, config: dict):
        """持久化任务配置"""
        path = self.tasks_dir / f"task_{task_id}.json"
        path.write_text(json.dumps(config, ensure_ascii=False, indent=2))

    def load(self, task_id: int) -> dict | None:
        path = self.tasks_dir / f"task_{task_id}.json"
        if path.exists():
            return json.loads(path.read_text())
        return None

    def remove(self, task_id: int):
        path = self.tasks_dir / f"task_{task_id}.json"
        path.unlink(missing_ok=True)

    def list_all(self) -> list[tuple[int, dict]]:
        """列出所有持久化任务（用于重启恢复）"""
        result = []
        for path in sorted(self.tasks_dir.glob("task_*.json")):
            task_id = int(path.stem.split("_")[1])
            config = json.loads(path.read_text())
            result.append((task_id, config))
        return result

    def exists(self, task_id: int) -> bool:
        return (self.tasks_dir / f"task_{task_id}.json").exists()
```

## 2. 断线重连策略

### 2.1 心跳重试

```
正常: 每 30s 心跳
失败: 1s → 2s → 4s → 8s → 15s → 30s → 保持 30s
恢复: 立即回到 30s 间隔
```

```python
class HeartbeatRetry:
    def __init__(self):
        self.delays = [1, 2, 4, 8, 15, 15, 30]
        self.attempt = 0

    async def wait(self):
        delay = self.delays[min(self.attempt, len(self.delays)-1)]
        self.attempt += 1
        await asyncio.sleep(delay)

    def reset(self):
        self.attempt = 0
```

### 2.2 重连恢复流程

```
网络恢复
  │
  ▼
Agent 心跳成功
  │
  ▼
Web 端：
  1. 更新盒子 status=ONLINE
  2. 对比 tasks 状态差异
  3. 下发差异命令
      - 本地运行但 Web 已停止 → stop_task
      - Web 标记运行但本地未起 → start_task
      - 一致 → 无操作
  │
  ▼
Agent：
  1. 同步本地任务（创建/删除）
  2. 开始补传离线期间缓存的结果
```

## 3. 离线结果缓存

### 缓存策略

```python
class ResultCache:
    """检测结果离线缓存"""
    def __init__(self, max_size: int = 1000, cache_dir: str = "/opt/ai-box-agent/cache/"):
        self.max_size = max_size
        self.cache_dir = Path(cache_dir)
        self.cache_dir.mkdir(parents=True, exist_ok=True)
        self._queue: list[dict] = []

    def push(self, detection_event: dict):
        """缓存一条检测结果"""
        self._queue.append(detection_event)
        # 超过上限丢弃最早的数据
        if len(self._queue) > self.max_size:
            self._queue.pop(0)

    async def flush(self, callback_url: str, callback_token: str):
        """批量回传缓存的结果"""
        if not self._queue:
            return
        batch = self._queue.copy()
        self._queue = []
        for event in batch:
            try:
                await httpx.post(
                    callback_url,
                    json=event,
                    headers={"Authorization": f"Bearer {callback_token}"},
                    timeout=10,
                )
            except Exception:
                # 重传失败重新入队
                self._queue.append(event)
                break  # 停止后续回传，下次重试
```

### 何时缓存

Worker 检测到事件后，回调分两步：

```python
# Worker 中
async def report_detection(event):
    try:
        await httpx.post(callback_url, json=event, timeout=5)
    except Exception:
        # 失败则写入本地 FIFO 文件
        result_cache.push(event)
```

### 何时刷新

Agent 检测到心跳恢复后：

```python
# Agent 心跳成功后
async def on_heartbeat_success():
    await result_cache.flush(config.callback_url, config.callback_token)
```

## 4. 启动恢复流程

```
Agent 进程启动
  │
  ▼
读取 config.yaml
  │
  ▼
注册（重试直到成功）
  │
  ▼
加载 /tasks/ 目录
  │
  ▼
遍历 task_*.json:
  for each task:
    if task_manager 中不存在:
      task_manager.start(task_id, config)
    # 任务已在运行 ↓
  │
  ▼
进入心跳主循环
```

## 5. 边界情况处理

| 场景 | 行为 |
|------|------|
| Agent 崩溃重启 | 恢复所有持久化任务，Worker 可能仍在运行（孤儿进程），清理后重新启动 |
| Worker 崩溃 | task_manager 监控 proc.poll()，自动重启（最多 3 次），超过则标记 ERROR |
| 磁盘写满 | 停止快照保存，告警日志，保留旧任务运行 |
| Web 主站迁移/更换 IP | 更新 config.yaml 中的 URL，重启 Agent |
| 盒子硬件更换 | 更换 box_id + secret_key，重新注册 |

## 6. 安全性考虑

| 风险 | 应对 |
|------|------|
| config.yaml 泄露 | 密钥存储在文件系统，限制权限 `chmod 600` |
| 中间人攻击 | 通信使用 HTTPS，Agent 验证证书 |
| 重放攻击 | Heartbeat 带 timestamp，服务端校验偏差 < 60s |
| 盒子冒名注册 | secret_key 预共享，不在网络中明文传输 |
