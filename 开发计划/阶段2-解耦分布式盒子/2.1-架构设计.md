## 与阶段1的差异

| 维度 | 阶段1（单机） | 阶段2（分布式） |
|------|--------------|----------------|
| Worker 位置 | Web 服务器本地 | AI 盒子远程 |
| 进程管理 | Web 内 Scheduler 直接管理 | Box Agent 本地管理 |
| 任务分配 | 本地 spawn | Web 下发指令 → Agent 执行 |
| 通信方向 | N/A（本地进程间） | 全部由 Agent 主动发起 HTTP |
| 离线韧性 | N/A | Agent 本地持久化配置 |
| GPU 资源 | 单机共享 | 每盒独立 |

## 网络架构

### 通信方式

**全部请求由 AI 盒子主动发起**（解决 NAT/防火墙穿透）：

```
 Agent ──HTTP──▶ Web 主站
 盒子主动请求    公网可达
 (从不反向)
```

### 跨网段弱网适配

| 机制 | 说明 |
|------|------|
| 心跳 30s | Agent 每 30s 发送心跳，Web 在响应中带回命令 |
| 命令轮询 | Web 不主动推，Agent 通过心跳轮询获取待执行命令 |
| 指数退避 | 网络失败时 1s→2s→4s→30s 递增重试 |
| 本地持久化 | 所有任务配置写入 `/etc/ai-box/tasks/`，重启恢复 |
| 结果缓存 | 回调失败时缓存检测结果，最多 1000 条 |

## 关键设计决策

| 决策 | 原因 |
|------|------|
| Agent 主动拉取命令 | 盒子在 NAT 后，Web 无法直达 |
| 每次心跳返回全量命令 | 避免 Agent 漏掉命令（网络丢包时下个周期必达） |
| 配置本地持久化 | Agent/盒子重启后自动恢复任务，无需 Web 重新下发 |
| callback 直连 Web | 直接复用阶段1的 callback 端点，Worker 不需感知 Agent 存在 |
