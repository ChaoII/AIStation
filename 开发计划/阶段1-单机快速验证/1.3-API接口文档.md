# 阶段1：单机快速验证 — API 接口文档

所有端点挂在现有 `AlgorithmRouter` 下（`prefix=/algorithm`），复用 `AuthPermission` 依赖。

## 1. 启动推理任务

```
POST /api/v1/algorithm/task/{id}/start
```

**权限：** `module_video:algorithm:update`

**路径参数：**
| 参数 | 类型 | 说明 |
|------|------|------|
| id | int | 算法任务 ID |

**响应：**
```json
{
  "code": 200,
  "data": {
    "task_id": 1,
    "status": "RUNNING",
    "pid": 12345,
    "message": "推理任务已启动"
  },
  "msg": "启动成功"
}
```

**错误：**
- 404: 任务不存在
- 409: 任务已在运行中
- 400: 算法模型文件不存在

## 2. 停止推理任务

```
POST /api/v1/algorithm/task/{id}/stop
```

**权限：** `module_video:algorithm:update`

**响应：**
```json
{
  "code": 200,
  "data": {
    "task_id": 1,
    "status": "STOPPED"
  },
  "msg": "停止成功"
}
```

## 3. 查询推理状态

```
GET /api/v1/algorithm/task/{id}/inference-status
```

**权限：** `module_video:algorithm:query`

**响应：**
```json
{
  "code": 200,
  "data": {
    "task_id": 1,
    "status": "RUNNING",
    "pid": 12345,
    "camera_name": "仓库_内部东",
    "algorithm_name": "入侵检测",
    "uptime_seconds": 3600,
    "started_at": "2026-05-31T09:00:00",
    "fps": 12.5,
    "stream_url": "http://...",
    "inference_latency_ms": 8.3
  },
  "msg": "查询成功"
}
```

**status 枚举：** `RUNNING | STOPPED | ERROR | UNKNOWN`

## 4. 检测结果回调（内部接口）

```
POST /api/v1/algorithm/detection/callback
```

**认证：** `Authorization: Bearer {shared_secret}`（非用户 JWT，通过配置共享）

**请求体：**
```json
{
  "task_id": 1,
  "camera_id": 24,
  "algorithm_type": "INTRUSION",
  "detections": [
    {
      "label": "person",
      "label_id": 0,
      "confidence": 0.92,
      "bbox": {"x": 0.1, "y": 0.2, "width": 0.3, "height": 0.5}
    }
  ],
  "snapshot_data": "/9j/4AAQ...base64...",
  "snapshot_path": "detections/2026-05-31/1_1717123456.jpg",
  "frame_timestamp": "2026-05-31T10:00:05.789Z",
  "inference_latency_ms": 12.3
}
```

**snapshot_data** 说明：base64 编码的 JPEG 图像（带标注框），由 service 层解码后写入磁盘。可选字段 — 如果 Worker 已经写入文件系统共享目录则传 `snapshot_path` 即可。

**响应：**
```json
{
  "code": 200,
  "data": {
    "alarm_id": 42,
    "alarm_created": true,
    "rule_matched": "入侵检测规则-仓库东"
  },
  "msg": "处理完成"
}
```

## 5. 保存检测快照（可选独立接口）

如果 Worker 和 Web 不在同一文件系统（阶段2 场景），快照通过回调中的 `snapshot_data` base64 传递，无需额外接口。

## 6. 配置常量

在 `settings.py` 中添加：

```python
# inference callback shared secret
INFERENCE_CALLBACK_TOKEN: str = "infer_shared_secret_change_me"

# worker snapshot directory
DETECTIONS_DIR: str = "data/detections/"
```
