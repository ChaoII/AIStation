## 组件职责

| 组件 | 职责 | 技术选型 |
|------|------|----------|
| **FastapiAdmin** | Web 主服务：API / WebSocket / 告警 / 通知 | FastAPI + Uvicorn |
| **Inference Scheduler** | Worker 子进程生命周期管理 | 模块内 asyncio 任务 |
| **Inference Worker** | 拉流 → 解码 → 推理 → 回调 | 独立 Python 子进程 |
| **ModelDeploy SDK** | ONNX/MNN/TRT 模型推理 | C++ Python Binding |
| **ZLMediaKit** | 摄像头流代理、协议转换 | 现有组件复用 |
| **PostgreSQL** | 业务数据持久化 | 现有组件复用 |
| **Redis** | 缓存 / 限流 | 现有组件复用 |

## 核心数据流

```
摄像头 RTSP
  │
  ▼
ZLMediaKit (addStreamProxy)
  │ FLV / RTSP
  ▼
Inference Worker (OpenCV VideoCapture)
  │ 每 N 帧:
  │   1. decode frame
  │   2. ROI masking
  │   3. model.predict(frame)
  │   4. filter by confidence
  │   5. alarm throttle check
  │   6. save annotated snapshot
  ▼
HTTP POST → /api/v1/algorithm/detection/callback
  │
  ▼
service.process_detection_event()
  │   1. validate token
  │   2. match AlarmRuleModel
  │   3. create AlarmRecordModel
  │   4. save snapshot to disk
  │   5. dispatch_notification()
  ▼
AlarmWSManager.broadcast() ── WebSocket ──▶ Frontend
```

## 设计原则

1. **子进程隔离**：每个算法任务运行独立 Worker 进程，崩溃不影响主服务
2. **无状态 Worker**：Worker 只读配置 → 拉流 → 推理 → 回调，不写数据库
3. **配置驱动**：Worker 启动参数由 JSON 配置文件指定
4. **现有设施优先**：告警创建、通知分发复用 AlarmService
5. **同步回调**：Worker 通过 HTTP POST 同步上传结果，简化 Worker 逻辑
